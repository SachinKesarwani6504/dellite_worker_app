UI CONSISTENCY RULES (USE FOR CUSTOMER APP)

GOAL
- Customer app should look like same product family as worker app.

LAYOUT RULES
- Use GradientScreen for page wrappers.
- Use APP_LAYOUT.screenHorizontalPadding for consistent page gutters.
- Keep cards rounded (mostly rounded-2xl or rounded-3xl).
- Keep spacing scale consistent (mt-2, mt-3, mt-4, mt-5 patterns).

THEME RULES
- Support light + dark with class names (dark:*).
- Use same brand tokens and text colors.
- Reuse shared theme utilities from src/utils/theme.ts style.

TYPOGRAPHY RULES
- Main page title: bold/extrabold, large display text.
- Subtitle: muted tone (#6E6E77 light, #B5B5BD dark).
- Labels and buttons: semibold/bold with clear contrast.

COMPONENT RULES
- Always prefer shared components:
  - Button
  - AppInput
  - BackButton
  - GradientScreen
  - GradientWord (where branding emphasis is needed)
  - BrandRefreshControl hook (for pull-to-refresh props)

INTERACTION RULES
- Use loading states for async actions.
- Disable buttons while submitting.
- Keep pull-to-refresh theme-aware and native.
- Hide BackButton when navigation cannot go back (visible={navigation.canGoBack()}).

COPY/CONTENT RULES
- Keep text tone friendly and action-oriented.
- Put all user-facing strings in one text constants file (APP_TEXT style).

DO NOT CHANGE
- Alias import pattern (@/)
- Folder architecture style
- Existing naming conventions for actions/types/hooks/navigation
