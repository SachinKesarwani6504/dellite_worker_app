CUSTOMER APP BUILD PLAN (SCALABLE)

TARGET
- Build dellite_customer_app using same code quality and UI consistency.
- Keep auth flow same base.
- Keep customer onboarding minimal and single-screen.

STEP 1: CLONE STRUCTURE
- Create same src folder tree as worker app.
- Copy reusable common UI components first.
- Copy theme, app text pattern, utils, icon wrapper, and navigation shell.

STEP 2: SET CUSTOMER ROLE DEFAULT
- In auth types/constants, set APP_AUTH_ROLE = 'CUSTOMER' for customer app.
- Keep API methods same style (/auth/send-otp, /auth/verify-otp, /auth/me).

STEP 3: CUSTOMER ONBOARDING FLOW
- Create one onboarding screen only:
  - fields: firstName, lastName
  - submit and complete onboarding
- Remove worker-only onboarding screens:
  - vehicle/service selection
  - certification requirements

STEP 4: NAVIGATION FOR CUSTOMER
- AuthNavigator: PhoneLogin, OtpVerification
- OnboardingNavigator: OnboardingCustomerIdentity, OnboardingWelcome (optional)
- MainTabsNavigator: customer tabs (home/orders/profile etc. as needed)

STEP 5: BACKEND FLAG MAPPING
- Use /auth/me response onboarding.CUSTOMER flags:
  - isBasicInfoCompleted => decide onboarding vs authenticated state
- Route decision should happen in AuthContext refreshMe flow.

STEP 6: UI CONSISTENCY CHECKLIST
- Same color tokens
- Same button/input style
- Same spacing rhythm
- Same dark mode behavior
- Same loading and toast behavior

STEP 7: TEST CHECKLIST
- Fresh install -> OTP -> customer onboarding -> app entry
- App reopen with token -> should skip completed onboarding
- Theme switch light/dark -> no visual break
- Pull-to-refresh pages should remain stable

OPTIONAL MIGRATION SHORTCUT
- Duplicate this worker app codebase.
- Rename worker-specific files/routes to customer equivalents.
- Remove worker-only API calls and screens.
